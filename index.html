    How to read this
    
      
    A single request travels a U-shaped path: it descends the client's stack getting wrapped (encapsulation), crosses the underlay network, then ascends the server's stack getting unwrapped (decapsulation).
    
      
      
        
    • Advance it by scrolling over the stage, dragging the scrubber, pressing ←/→, or hitting Space to auto-play.
      • 
        
    • Signal purity shows how little of the packet is real application data. Hit ↻ New packet in Explore mode to see it swing with payload size.
      • 
        
    • Highlight readers dims everything a given component can't see — proof that each layer is a blind courier reading only its own envelope.
      • 
        
    • Tool output reveals the real tcpdump / ss / ip you'd run at each step.
      • 
        
    • Diagnose mode plays a real production failure and asks you to spot the breaking layer.
      • 
      
    
      
    Layer model: we use the practical TCP/IP view — Application (HTTP, with TLS as an optional wrapper) over Transport (TCP) over Network (IP), plus the Overlay (VXLAN) tunnel. No OSI L5/L6 numbering, because in the cloud nobody debugs "a Layer 6 problem" — they debug a TLS handshake.
